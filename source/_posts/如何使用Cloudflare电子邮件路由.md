# 一、准备工作
* 购买域名：作为之后电子邮箱路由的后缀
* 托管至Cloudflare：参考之前的文章即可

# 二、配置 Cloudflare 邮件路由
## 1. 进入邮件路由配置页面

在 Cloudflare 控制台，进入你想要作为域名后缀的域名管理页。
点击左侧**电子邮件**→**电子邮件路由**。

## 2. 添加目标地址（代收邮箱）

切换到**目的地地址**标签页，点击**添加目的地地址**。
在输入框中填入你的主邮箱(如Gmail邮箱)，点击**保存**。
Cloudflare 会向该邮箱发送一封验证邮件，登录邮箱并点击验证链接，完成所有权验证。

## 3. 创建路由规则（对外邮箱）

切换到**路由规则**标签页，点击**创建地址**。
配置规则信息：
* 自定义地址：输入你想要的邮箱前缀，后缀选择你的域名
* 行动：选择发送邮件
* 目的地：选择刚才验证的主邮件地址
点击**保存**。

## 4. 验证配置状态

回到**电子邮件路由**概览页，查看刚才添加的路由的状态。
当状态从**同步中**变为**已启用**，且 DNS 记录显示为**已锁定**时，说明配置已完成。

# 三、测试邮件代收功能
使用任意邮箱（不得为主邮箱）给对外邮箱发送一封测试邮件。
登录你的主邮箱，检查是否收到该邮件。
若收到，说明邮件代收功能已正常工作。
