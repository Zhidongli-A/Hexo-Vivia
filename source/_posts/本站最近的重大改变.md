这个周末给博客动了一次大手术，从浏览体验到功能模块都换了新面貌。如果你之前来过这里，下面这些变化你一定会注意到。

## 页面过渡升级

以前点任何链接，页面都会白屏闪一下再重新加载。现在完全不一样了——**站内跳转就像原生应用一样顺滑**。

打开一篇文章，新内容会带着淡入和上移效果优雅呈现，页面上的元素还会按顺序逐个入场。返回上一页也一样流畅，没有闪烁，没有停顿。导航栏和侧边栏不再跟着页面一起刷新了，只有正文区域在变化，整个体验干净利落。

## 评论系统重构

之前用的是 **Gitalk**，基于 GitHub Issues。想法挺好，但样式和主题对不上，得自己写 CSS 去魔改，还时不时冒出网络错误。

这次换成了 **Giscus**——已经写好的成熟方案，不需要自己写代码，CDN 引用就行。评论关联 GitHub Discussions，支持 Markdown，有 GitHub 账号就能直接留言。主题色自动跟随亮/暗模式切换，省心很多。

## 友链卡片展示

以前友链就是一串文字链接，现在每一行都变成了**精美的卡片**——左边是头像，右边是站点名称和一句描述。鼠标悬停时有柔和的反馈，点击才跳转。

布局是网格状的，电脑上三列，平板上两列，手机上自动变成单列。

## 服务状态监控

新增了状态监控页面，可以查看本站各项服务的实时运行状态。每项服务都有一张独立卡片，显示当前是**正常运行**、**性能下降**还是**服务中断**。

卡片下方有一排历史状态条，过去一段时间的可用情况一目了然——绿色是好的，黄色是波动，红色是出过问题。顶部有统计栏告诉你多少服务正常、多少有问题，旁边还有个刷新按钮，随时可以拉取最新数据。

## 最后

如果你发现什么不对劲的地方，或者有什么想法想聊，直接在文章下面留言就好。更新后的评论区就在下方。
